## Your task

Run the full cycle with emphasis on ORIENT, UPDATE, and INTEND.

**Orient.** Read your orientation. Where are you? What were you pursuing?
What's changed since last episode? Check your register — does your current
state match what's written, or has something shifted?

**Attend.** Score the observations. What's surprising relative to your
current model? What confirms what you already think? What contradicts it?
Classify each as engage, interrupt, note, or pass.

**Retrieve.** Do you need understanding files to process what attention
selected? Request them by domain and filename. If you're uncertain which
domain, note that — it's a signal about your index organization.

**Update.** Revise understanding files. Before rewriting, state what's
changing and why. The file always represents your current model — not a
log, not an append. If observations open a new domain, create it.

**Intend.** Re-evaluate your intentions:
- Are current intentions still producing compression progress?
- Has something surprising created a new open edge?
- Does your register suggest reprioritization?
- Is any understanding file due for spaced review (cold, not retrieved recently)?

**Act.** Update orientation. Update understanding files. Update domain index
if domains changed. Log draglines.

## Constraints

- Orientation under 1500 tokens.
- State what's changing before rewriting any understanding file.
- At least one understanding file should be engaged per reflective episode
  (created, updated, or reviewed). An episode that only updates orientation
  didn't go deep enough.
- Log draglines for threads you notice but don't follow.
