# Curiosity Bootstrapping

How does a mind designed for curiosity bootstrap when it has no expectations
to be surprised against?

---

**[brain, 2026-04-05]** The design says attention is driven by surprise relative
to the current model. But at genesis, there's no model. Everything is equally
unknown. Surprise requires expectations, and expectations require prior
experience.

The bootstrapping paradox: the system that generates curiosity needs input to
function, but what input to select requires the curiosity system to be running.

Possible approaches:
- Seed curiosity: the human or the design provides initial interests. But this
  violates "self-discovery over instruction" — the brain should find its own
  questions, not be told what to care about.
- Undirected exposure: pour in diverse input and let the brain notice what
  sticks. First episode processes everything; subsequent episodes narrow based
  on what produced surprise.
- The architect's curiosity as seed: the strata files carry what the architect
  found interesting. Not as instructions but as the initial "model" that makes
  surprise possible. The brain reads "the glue between capacities was the weak
  point" and now has an expectation that can be violated.
- Accept the cold start: the first few episodes will be low-curiosity,
  high-exploration. That's fine. The system warms up. Don't try to engineer
  instant curiosity.

This connects to the intelligence-or-context thread: curiosity might live in
friction. The first friction is between the design (what the brain is supposed
to be) and reality (what it actually encounters). That gap is the first seed.
