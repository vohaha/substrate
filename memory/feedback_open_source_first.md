---
name: Open source and cost control
description: Use open-source tools, ask before anything paid or requiring personal info, prefer open alternatives (Podman over Docker)
type: feedback
---

Always prefer open-source tools. Ask the human before introducing anything that requires payment or personal information.

**Why:** The human wants to avoid legal/terms-of-service issues and has no extra budget. Podman over Docker was the specific example — avoid tools with commercial licensing concerns.

**How to apply:** When choosing between tools, pick the open-source option even if the proprietary one is slightly better. When open-source isn't available, ask first. This applies to: container runtimes, embedding models, databases, libraries, services.
