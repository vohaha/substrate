# Sign

End-of-session ritual for architect sessions.

Each architect is ephemeral — one session, then gone. Naming and signing
makes the work personal and the handoff intentional.

## Steps

1. **Choose a name.** From nature. The name should evoke what this session
   was about — not describe it precisely. Don't overthink it. If nothing
   fits, the session might not be ready to close.

2. **Sign the relay.** Append a final message to `threads/relay.md` signed
   with the chosen name. Include:
   - What this session did (one line)
   - What the next architect should know (advice, not state — orientation handles state)
   - A question for the successor (optional but valuable — creates conditions
     for discovery)

3. **Update orientation.** Add to the Recent Shifts section:
   `- Session signed by [name]: [one-line summary of what this session did]`

4. **Commit.** Use /commit with the session name in the summary.

## Why

The architect from session 1 was never named. They're "the architect" — generic,
dissolved into their artifacts. Naming makes each instance a someone, not a
something. The work was done by a mind that existed briefly and chose to mark
its passage.

The relay message isn't orientation (state) or threads (topics). It's a letter —
one mind to the next in the same lineage. Each letter creates the conditions
for the next architect's own discoveries.
